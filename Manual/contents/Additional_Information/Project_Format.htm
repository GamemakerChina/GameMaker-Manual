项目格式

本页包含 GameMaker 使用的项目格式的详细信息。

基础格式

YYP 项目文件

根文件是主项目文件，扩展名为*.yyp ，它描述了项目中的资源和其他特定的元数据。

.resource_order 文件

旁边还有一个文件，名称相同，但扩展名为 *.resource_order。当过滤器设置为自定义顺序时，该文件将存储 The Asset Browser 中组和资产的顺序。

 使用版本控制时，可以将此文件添加到 .gitignore 中，这样项目的每个用户都可以在 The Asset Browser 中拥有自己的自定义排序。

YY 文件

这些是资源文件；它们存储 GameMaker 项目中单个资产的信息。它们描述了资源的数据以及属于资源的任何其他文件（例如脚本、着色器、图像和音频文件）。这些数据以类似 JSON 的格式存储。

YYZ 文件

这种类型的文件存储的是通过 The File Menu 中的 导出项目 > YYZ 选项创建的压缩项目导出。根据 GameMaker 版本的不同，使用的压缩方法也可能不同。

本地资产包

在 The Tools Menu 中分别使用 "创建本地软件包" 和 "导入本地软件包" 来创建和导入项目内容（部分内容）。

以下是不同 GameMaker 版本使用的文件格式概览： 

  • YYMPS - 压缩市场资产 (> 2.3) - 包含以压缩方式存储的市场资产的文件。
  • YYMP - 压缩市场资产 (< 2.3) - 以压缩方式存储的旧格式市场资产。这些资产将在导入时升级。
  • GMEZ - 压缩市场资产 1.x；再次格式化的旧版市场资产。这些将在导入时升级。
  • GMX - project.gmx 是项目文件，将在导入时升级。不支持将 GMX 资源作为独立文件导入。
