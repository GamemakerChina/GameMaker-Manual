运行程序错误

即使在代码编辑器中进行语法检查，然后让编译器检查代码是否有编译器错误，仍有可能出错。在大多数情况下，这会产生一个 虚拟机 (VM) 运行程序错误（也称为运行时异常），看起来像这样：

运行错误通常比编译错误更严重，因为它意味着代码中存在代码编辑器和编译器都无法检测到的严重错误，因此你应该注意所有此类错误。出现错误时，您可以使用弹出窗口上的Copy按钮将错误复制到剪贴板，然后粘贴到文本文件（或其他地方）中，以备将来参考。

  • 这显示了实际错误。 因此，它显示了错误的范围和错误的类型。括号中的第一个值将是运行代码的实例的实例 ID，也可能是下表中列出的特殊值之一，第二个值是 GameMaker 用于识别错误的内部值，可以忽略。在上面的图片中，它表示对象 obj_EA_DEMO_Control 上的变量 _image 未被设置（即：在被访问前不存在）。

如上所述，某些错误信息不是通过实例 ID 值来标识作用域，而是通过负值来标识。这些值可用来确定错误的确切性质以及它所指的是什么，可能的值如下：

前缀 范围
-1 Self
-2 Other
-3 All
-4 Noone
-5 Global
-6 Not Specified
-7 Local

 

虚拟机运行程序可能出现的错误如下：

错误 信息 类型 说明
DoSet Invalid comparison type Data Types 这表示运行程序尝试比较两种不兼容的数据类型，如实数和字符串。
DoConv Execution Error Data Types 这表示一种数据类型转换成另一种数据类型时出现错误。
Argument Type (function) argument (index) incorrect type (wrong_type) expecting a (expected_type) Data Types 将一个数据类型错误的值作为参数传递给了函数 (index)（其中索引 1 是第一个参数）。给出的值类型是 wrong_type，而应该是 expected_type。
DoAdd Execution Error Maths 使用加法 (+) 表达式时出了问题。
DoMul Execution Error Maths 使用乘法 (*) 表达式时出了问题。
DoSub Execution Error Maths 使用减法 (-) 表达式时出了问题。
DoSub Execution Engine - Cannot operate on string type Maths 您试图减去错误类型的变量（例如从实数减去字符串）。
DoDiv Execution Error Maths 使用除法 (/ 或 div) 表达式时出了问题。
DoDiv Execution Engine - Cannot operate on string type Maths 您正在尝试除以错误类型的变量（例如用字符串除以实数）。
DoDiv Divide by zero Maths 您正在尝试除以 0（注意，只有使用整数除法时才会出现这种情况，用 0 除以（非 0）实数的答案是无穷大，用 0 除以 0 的答案是 NaN）。您可以使用 (is_infinity) 和 (is_nan) 检查这些值。
DoMod Execution Error Maths 使用求余运算（mod）表达式时出了问题。
DoMod Execution Engine - Cannot operate on string type Maths 您试图在错误类型的变量上使用求余运算 (mod)（例如用实数模字符串）。
DoAnd Execution Error Bitwise 使用位运算 "and"（&）表达式时出了问题。
DoAnd Execution Engine - Cannot operate on string type Bitwise 您尝试在错误类型的变量上使用位和 (&)（例如，尝试将字符串与实数 "和 "在一起）。
DoOr Execution Error Bitwise 使用比特 "or"(|) 表达式时出了问题。
DoOr Execution Engine - Cannot operate on string type Bitwise 您尝试在错误类型的变量上使用 "or"（|）（例如，尝试将字符串与实数 "or"）。
DoXor Execution Error Bitwise 在使用比特 "xor" (^)表达式时出了问题。
DoXor Execution Engine - Cannot operate on string type Bitwise 您尝试在错误类型的变量上使用 "xor" (^)（例如，尝试将字符串与实数 "xor"）。
DoShl Execution Error Bitwise 向左位移 (<<) 数值时出了问题。
DoShl Execution Engine - Cannot operate on string type Bitwise 您正在尝试对错误类型的变量进行左位移 (<<)（例如尝试对字符串进行位移）。
DoShr Execution Error Bitwise 向右位移 (>>) 数值时出了问题。
DoShr Execution Engine - Cannot operate on string type Bitwise 您正在尝试对错误类型的变量进行右位移 (>>)（例如尝试对字符串进行位移）。
DoNeg Execution Error Negate 您试图将一个变量类型转化为负数，而该类型不允许这样做。
DoNot Execution Error Negate 您正在尝试 "not" 变量类型，而该类型不允许这样的操作。
Push Execution Error - Variable Index out of range (var) Stack 访问的变量超出了运行程序的既定范围。
Push Variable Get (var) Stack 给定变量未定义或未知。
Pop Variable Index out of range (var) Stack 访问的变量超出了运行程序的既定范围。
Pop Variable Get (var) Stack 给定变量未定义或未知。
With Cannot use global in with statement With 您尝试在 "with "语句中使用 "global "作为变量，即
with (global)
   {
   //do something;
   }
With Cannot use local in with statement With 您尝试在 "with "语句中使用 "local "作为变量，即
with (local)
   {
   //do something;
   }
DoCall Execution Engine type error Engine 这是虚拟机内部未定义的错误。如果出现这种情况，您应该提交错误报告（有关如何提交错误报告的详细信息，请参阅：帮助菜单。
Stack Overflow - Engine 堆栈溢出发生在调用堆栈使用了太多内存的情况下，当你的游戏试图使用超过调用堆栈可用空间的空间时（也就是说，当它试图访问超出调用堆栈边界的内存时，本质上就是缓冲区溢出），堆栈就会溢出，导致程序崩溃。重新启动电脑和 GameMaker，如果错误仍然存在，请联系技术支持和/或提交错误报告（如上所述）。
Read Variable Variable not set before reading it Variable Initialisation 您正在尝试访问一个尚未设置（即初始化）的变量。在尝试读取之前，请先为变量赋值，例如 variable = 100;。只声明一个变量，例如使用 var variable; 也会产生此错误。
