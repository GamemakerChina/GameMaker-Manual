编译器错误

游戏报告的第二类错误是编译器错误。编译器错误发生在游戏遇到语法检查器无法检测到的错误时 - 例如，忘记声明变量或错误地将字符串键入为实数 - 或者错误与您在 偏好设置 中为目标平台设置编译工具的方式有关。GameMaker 会在编译游戏时捕捉到这些错误，这些信息也会显示在 编译器错误输出 中，如下图所示：

注意 这些错误通常不会弹出窗口，只会在编译器输出中显示 "Build Failed（编译失败）"。如果弹出窗口，通常是运行程序错误，此处有解释。

编译器错误信息的格式都是一样的：

[对象] - [事件] - [行号]: [错误字符串］

如果在脚本中发现错误，那么就是：

[脚本] - [行号]: [错误字符串]

您可以双击任何一个编译器错误条目，在标记为出错的位置打开给定的资产，然后您就可以使用消息中包含的信息来准确定位错误发生在对象或脚本的哪个位置。不过，有时这些错误看起来有点莫名其妙，因此下面列出了所有错误的完整列表，并简要解释了这些错误的含义：

Unable to find function [function]     您调用了未知函数

Calling a function that needs an other and no other is available 您在碰撞事件之外或 with 语句之外使用了关键字 other
Argument naming error, n arguments but no reference found to a 您为脚本提供了多个参数，但却没有在脚本中引用其中一个参数，例如，您的脚本需要三个参数，但您只使用了参数 n 和 a
Cannot compare arguments 您尝试比较两种不同的参数类型，如字符串和实数

No references to argument[num] but references argument[num] 您正在使用对一个不存在的参数的引用，但该参数可以映射到另一个参数
Incorrect number of arguments for accessor 您在使用访问器时提供了错误的参数数
Wrong type of arguments for (one of the below):
/, div, mod, +, -, &&, ||, &, |, ^, <<, >>		 您为正在求值的表达式提供了错误的参数类型
