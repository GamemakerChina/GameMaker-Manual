语法错误（羽毛信息）

在 GameMaker 中进行游戏编程时，第一级错误报告在代码或 GML Visual 编辑器中，这就是 语法 检查器。当你编写代码或添加操作时，GameMaker 会检查你的代码是否遵循了既定的语言语法，并标记出发现的任何问题，以便你在运行游戏或编译可执行文件前解决这些问题。

语法错误检查会以不同的方式发生，这取决于您是否启用了 羽毛 在 羽毛设置 当中。

语法错误（禁用羽毛）

这些错误主要分为两大类：

  •  警告: 这些警告显示的内容可能是错误，但也可能不是；可能不会阻止游戏编译，但可能会引发运行时错误。这些警告最常见的原因是表达式中使用了一个变量，但该变量尚未在其他地方定义，或者变量已经定义但尚未使用。
  •  错误: 这些问题会导致游戏无法编译，更不用说运行了。在进一步处理代码之前，应先解决这些问题。下表显示了最常见的原因。

语法错误将在您编写代码时更新（在输入内容和错误出现在此窗口之间会有短暂的停顿，以防止未完成的代码被报告错误），并将遵循格式：

[对象] - [事件] - [行号] - [行中位置]： [错误字符串］

或者如果错误出在脚本中，就会出错：

[脚本] - [行号] - [行中位置]： [错误字符串］

双击输出控制台中显示的任何错误，就会跳转到对象事件或脚本中出现错误的那一行，以便进行编辑。

需要注意的是，其中一些错误会显示在代码中的特定位置，但实际错误可能是由其他地方引起的，或者一个问题可能会引发多个错误信息：

语法错误列表

下表列出了不同的语法错误及其主要原因：

 

错误  类型  说明
Malformed reference [val] 这意味着出现了内部 AST 验证错误（您不应该看到这个错误，但如果您看到了，请考虑从帮助菜单中提交错误报告）
Malformed hexadecimal character escape sequence  如果您在字符串中使用了转义字符的十六进制值，而该值的格式有误，就会出现这种情况。
Unexpected node [val] 这意味着出现了内部 AST 验证错误（您不应该看到这个错误，但如果您看到了，请考虑从帮助菜单中提交错误报告）
Exception while parsing [val] 这意味着出现了内部 AST 验证错误（您不应该看到这个错误，但如果您看到了，请考虑从帮助菜单中提交错误报告）
Malformed id reference [val]  
Unnecessary expression [val] used as a statement 当语法检查程序检测到一个表达式（如 [val] 所示），而它本应检测到一个语句时，就会发生该错误。例如，下面的代码就会出错，因为它期待的是三元运算符 ? 却遇到了 32：
temp_x = x < (room_width / 2)  32: room_width - 32;
Malformed array reference [val] 这意味着您尝试以不正确的方式引用数组，[val] 将显示该引用。
Unexpected binary operator [val] 如果在不应该使用位运算符的地方使用了其中一个位运算符，就会出现这个错误。
Unterminated string literal 如果使用 @" 或 " 打开字符串，但没有在代码结束前关闭字符串，则会出现此错误。
Single quotes no longer allowed for string 当您尝试使用单引号包裹的字符串（如："Hello World"）而不是双引号包裹的字符串（如："Hello World"）时，就会出现此错误： "Hello World"。
Unexpected syntax error 该错误表示语法错误，解析器无法更具体地识别。
Got "[val1]", expected "[val2]" 当语法检查程序期望某个值或符号，但得到的却是不同的值或符号时，就会出现这种情况，其中 [val1] 是当前的符号或值，而 [val2] 是期望的符号或值。例如
得到"{"，预期为"("
Got "[val1]", expected "[val2]" or "[val3]" 同上，但适用于有多个可能的预期符号的情况。
Expected expression 当语法检查程序期望使用表达式但没有给出表达式时，就会出现该错误。
Empty [val] statement 当您终止语句并将其留空时，就会出现这种情况。例如，以下代码就会出错（注意终止分号）: 
if (x < 0);
Nested function calls are not allowed 当您尝试在语句中嵌套函数调用时，例如：
ds_list_find_value(list, 0)).sprite_index
Number of arguments for function [val1] expected [val2] got val[3] 该错误说明您没有为 [val1] 所示函数提供正确的参数数，该函数期望得到 [val2] 所示的参数数，但却得到了 [val3] 所示的参数数。
Number of arguments for function [val1] expected [val2] - [val3] got [val4] 同上，只针对一系列参数。
Duplicate enum entry found 该错误说明您重复了一个枚举变量条目名称 
Duplicate enum found 该错误说明您定义了两个或多个同名枚举。
Missing variable name in globalvar 这说明您使用了 globalvar 声明，但忽略了提供变量名。
Missing variable name in var 这说明您使用了 var 声明，但忽略了提供变量名。
No matching #region found for #endregion 该错误表示您在代码中声明了结束区域，但没有定义开始区域。
Unclosed #region found at end of script 该错误表示您在代码中的某处声明了一个区域，但没有为其定义结束区域。
Unexpected terminal operator [val] 这意味着出现了内部 AST 验证错误（您不应该看到这个错误，但如果您看到了，请考虑从帮助菜单中提交错误报告）
Unexpected unary operator [val] 这种情况发生在代码中发现了一个一元运算符（如 +、-、= 等......），而该运算符并不在预期之内，此时 [val] 会向您显示该运算符。
Unexpected ternary operator [val] 该错误表示您在代码中的某个地方错误地使用了三元运算符的某个部分，[val] 表示出错的部分。
Try needs to have catch or finally clause 该错误表示您调用了 try 函数，但忽略了 catch 或 finally 子句。
Macro [val] already exists 当您尝试定义与先前定义的名称相同的 [val] 宏时，可能会出现此错误。
Malformed variable reference, got [val] 这意味着出现了内部 AST 验证错误（您不应该看到这个错误，但如果您看到了，请考虑从帮助菜单中提交错误报告）
Assignment to multi-relational-equality expression - GML does not support multiple assignments in an expression 此错误说明 GameMaker 语言不支持多操作符赋值（也称为复合赋值）。这包括 "*="或"/="等。
Macro [val] is unused 如果代码中没有使用 [val] 宏，就会出现这种情况。
Variable [val] only referenced once 这意味着给定变量 [val] 在代码中只被引用一次。
Unassigned variable [val] referenced 这意味着 [val] 变量已在代码中被引用，但之前未在任何地方赋值。
Only functions that are declared as constructors can use inheritance 这个错误告诉你，你试图在一个未声明为构造函数的函数上使用继承。
Unknown function attribute [val] 该错误表示您在声明内置函数时尝试使用了无效或未识别的函数修饰符（[val]）（函数修饰符类似于构造函数关键字，是受支持的）。
Inherited argument \"[val]\" is not in function arguments 当您尝试在继承函数中引用父函数定义中未声明的参数时，会发生此错误。
[val] Creation Code 如果房间创建代码中的某个地方出现 [val] 错误，就会出现这种情况。

羽毛信息（启用羽毛）

启用 Feather 后，语法错误窗口将被羽毛信息窗口取代： 

羽毛消息窗口列出了羽毛在您的代码中遇到的错误和警告，以及它在发现某些特定代码模式时给出的建议。这里显示的错误类型取决于您在 羽毛设置 中的 "消息严重性 "下定义的配置文件。

在该窗口中，可按信息代码、信息文本（按字母顺序）或资源（按资源名称的字母顺序）对信息列表进行升序或降序排序。您还可以使用窗口右上角的搜索框进行搜索。

羽毛信息可分为三类：

  •  警告: 这些警告显示的是可能出现的错误，在大多数情况下，这些错误不会阻止游戏的编译，但可能会引发运行时错误。导致这些警告的最常见原因是表达式中使用了变量，而该变量尚未在其他地方定义，或者变量已经定义但尚未使用。
  •  错误: 这些问题会导致游戏无法编译，更不用说运行了。在进一步处理代码之前，应先解决这些问题。
  •  建议: 这些都是对代码可能的改进建议，涉及方方面面。
