代码注释

当与其他人协作时，当处理大型项目时，甚至只是为了自己参考调试时，在代码中留下注释是非常重要的。在 GameMaker 中，你有许多种方法可以帮助你在代码中留下有关当前代码段，甚至是整个代码块的笔记和注释，这些笔记和注释可以在当前正在编辑的物体的事件列表中查看。通过这种方式，你可以给自己或同事留下备注，或是解释一段特别复杂的代码，甚至只是留下一个提醒用来说明某些东西的作用，又或者是一些未来要做的东西

想要简单的留下一行注释，你唯一要做的事就是将 // 添加到文字前面，例如：

// 初始化变量
sr = 10;
hp = 100;

你还可以通过留下多行注释来展示感谢名单，或是调试时忽略某些代码段，或者记录你未来可能对代码进行的修改。这种多行注释可以使用 /* ... */ 来表示，就像这样：

/*
将GIF的变量设为初始值（这些可以被编辑，因此如果你总是使用相同的值，就不需要一直使用设置函数）

重点！请参阅各个"设置"脚本以获取有关每个不同变量的更多详细信息。
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

除了注释之外，你还可以使用特殊的 #region 标签来标记代码的一个部分，这可以用于代码折叠，并且每个区域都可以包含注释。其使用方式是，在代码的一个部分顶部添加一个 #region 标签 - 以及任何注释文本 - 然后在该部分的底部添加一个相应的 #endregion 标签，之后你可以“折叠”代码，即：隐藏它，这样你就可以减少在编辑器中一次可见的代码行数：

实际的代码看起来像这样：

#region 这是一个示例区域

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion 如有需要，此处可以进一步注释

最后，值得一提的是，在使用 物体编辑器 中的事件时，你可以在事件的开始使用一个特殊的“description”注释来说明事件的一般内容，这将显示在物体编辑器的事件窗口中。这应该放在代码编辑器的第一行，格式如下：

/// @description <这是事件描述>
