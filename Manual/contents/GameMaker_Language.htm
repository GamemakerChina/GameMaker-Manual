GameMaker Language

你可以使用 GameMaker Language 通过两种完全不同的方式来编写程序：通过简单的拖动积木块来“写”代码，亦或是手动的编写代码。

你的第一个选择，GML Visual，最适合想要快速开始制作自己的游戏的初学者和爱好者。

还有第二个选择，GML Code，它允许你通过手动编写代码来利用 Gamemaker 的所有功能。

虽然从 GML Visual 开始更加的容易，但是 GML Code 也是一种非常易于使用的编程语言，GML Code 具有强大的功能 (函数、方法、结构 & 构造函数等).

创建 Visual（可视化）/Code（代码） 脚本

在创建新物体的事件、新的脚本资源或是进行其他任何会创建脚本的行动时，系统会要求你在 GML Code 和 GML Visual 之间做选择:

除了 GML Code 和 GML Visual 之间的选择之外，这个窗口还包括两个复选框：

  • 不再询问此项目：启用此选项后，会将 游戏选项 中的 “默认脚本语言” 设置为你的选择，因此在当前项目中不会要求你再次选择
  • 不在询问：启用此选项后，会将 偏好设定 中的 “默认脚本语言” 设置为你的选择，因此对于所有项目都不会要求你再次选择
